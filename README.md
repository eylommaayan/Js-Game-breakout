פרויקט זה נוצר כחלק מחוג תכנות לילדים ונוער, במטרה ללמד את יסודות ה-JavaScript דרך בנייה אקטיבית של משחק קלאסי.

## 🎓 המתודולוגיה הלימודית: "להבין, לא רק להעתיק"
בניגוד לשימוש ב-AI ליצירת קוד אוטומטי, בחוג שלנו אנחנו מפרקים את המשחק לאתגרים לוגיים קטנים. המטרה היא שהתלמיד יבין כל שורה ושורה, כך שיוכל לשלוט בקוד ולשנות אותו כרצונו.

### שלבי הלמידה בפרויקט:
1. **הקנבס (The Canvas):** הבנת מערכת הצירים הגרפית וציור צורות בסיסיות.
2. **לולאת המשחק (Game Loop):** יצירת תנועה על ידי עדכון פריימים.
3. **לוגיקת התנגשויות (Collision Detection):** שימוש בתנאים (`if/else`) כדי לגרום לכדור להגיב לסביבה.
4. **ניהול מצבים (Game State):** יצירת משתנים בוליאניים לניהול מצבי "הפסד", "ניצחון" ו"הפסקה".

---

## 🚀 תכונות המשחק (Features)
* **מערכת ניקוד:** מעקב בזמן אמת אחרי שבירת לבנים.
* **רמות קושי משתנות:** מהירות הכדור עולה בכל פעם שהלוח מתנקה.
* **מסך Game Over:** עצירה מסודרת של המשחק ואפשרות להתחלה מחדש.
* **עיצוב רספונסיבי:** שימוש ב-CSS מודרני לחוויית משחק חלקה.

---

## 🛠️ טכנולוגיות
* **HTML5 Canvas** - לרינדור הגרפיקה.
* **CSS3** - לעיצוב הממשק ומסכי ההוראות.
* **Vanilla JavaScript** - ללא ספריות חיצוניות, כדי להבין את ה-"Core" של השפה.

## 👨‍🏫 משימות אתגר לתלמידים (Challenges)
כחלק מהחוג, התלמידים מתבקשים לבצע את השינויים הבאים בעצמם:
- [ ] **אתגר הצבעים:** שנה את הקוד כך שכל לבנה תקבל צבע רנדומלי בכל פעם שהיא נוצרת.
